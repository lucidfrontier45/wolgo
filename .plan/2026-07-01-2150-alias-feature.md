# Goal

Add alias registry for MAC targets, plus alias-aware wake/find-ip flows and `--all` fan-out.

# Background

Current CLI supports:
- `wolgo <MAC_ADDRESS>` → send WOL
- `wolgo find-ip <MAC_ADDRESS>` → resolve IPs from ARP cache

Requested additions:
- new subcommands:
  - `register <alias> <mac addr>`
  - `list`
  - `remove <alias>`
- new behavior:
  - `wolgo <alias>` resolves alias to MAC, then wakes target
  - `wolgo find-ip <alias>` resolves alias to MAC, then runs lookup
  - `--all` for wake and find-ip over all registered MACs
- storage path: `~/.wolgo/targets.json`
- conflict rule: if arg is valid MAC, treat as MAC, not alias
- list format: `alias -> mac`, sorted by alias
- `register` overwrites existing alias
- alias naming rules:
  - no spaces
  - no tabs
  - must not itself be valid MAC address

# Approach

1. Add alias store layer
   - Create helper(s) in `cmd/` to resolve `~/.wolgo/targets.json`
   - Load/store alias data as JSON
   - Ensure `~/.wolgo` exists before write
   - Return empty registry cleanly when file absent

2. Define registry format and validation
   - Store alias→MAC mapping in JSON
   - Validate on register:
     - alias non-empty
     - alias contains no spaces/tabs
     - alias is not parseable as MAC
     - MAC is valid
   - Normalize MAC before storing for consistent comparisons/output

3. Add shared target resolution helpers
   - Resolve single input by rule:
     1. if valid MAC, use MAC
     2. else look up alias
     3. else error
   - Add helper to enumerate all registered targets for `--all`
   - Keep root wake and `find-ip` behavior consistent

4. Extend root wake command
   - Add `--all` flag
   - Support:
     - `wolgo --all`
     - `wolgo <MAC_ADDRESS>`
     - `wolgo <alias>`
   - Reject invalid combinations like positional arg plus `--all`
   - For `--all`, iterate registered MACs and send WOL to each

5. Extend `find-ip`
   - Add `--all` flag
   - Support:
     - `wolgo find-ip --all`
     - `wolgo find-ip <MAC_ADDRESS>`
     - `wolgo find-ip <alias>`
   - For `--all`, output one sorted line per registered alias in exact form:
     - `alias -> mac`
   - Internally still perform IP lookup per target, but output remains literal alias/MAC list per requirement unless later expanded

6. Add new subcommands
   - `register <alias> <mac>`
     - validate inputs (alias first, MAC second)
     - normalize/store MAC
     - overwrite existing alias entry
     - print confirmation
   - `list`
     - load registry
     - sort aliases
     - print `alias -> mac`
   - `remove <alias>`
     - delete alias
     - error if alias absent

7. Update CLI docs/help
   - Update root usage/help text for alias support
   - Add examples for:
     - `wolgo register office-pc 00:11:22:33:44:55`
     - `wolgo office-pc`
     - `wolgo --all`
     - `wolgo find-ip office-pc`
     - `wolgo find-ip --all`
     - `wolgo list`
     - `wolgo remove office-pc`

8. Add tests
   - registry load/save with temp HOME
   - register overwrite behavior
   - alias validation rejects spaces/tabs/MAC-like names
   - resolution precedence: MAC beats alias
   - list sorting
   - remove missing alias error
   - `--all` target enumeration behavior

9. Validate
   - `go test ./...`
   - `go mod tidy`
   - `golangci-lint fmt`
   - `golangci-lint run`

# Trade-offs

- JSON file in home dir
  - pros: simple, portable, no new deps
  - cons: limited concurrency guarantees
- MAC normalization on write
  - pros: consistent storage/output, simpler comparisons
  - cons: original input formatting lost
- `find-ip --all` literal alias/MAC output
  - pros: matches stated requirement exactly
  - cons: surprising name/result mismatch since command no longer shows IPs in this mode

# Open questions

- None blocking. Only possible follow-up: whether `find-ip --all` should later include IP data under each `alias -> mac` line.

# Next step

Implement registry helpers first, then wire root command and `find-ip` to shared alias/MAC resolution.
